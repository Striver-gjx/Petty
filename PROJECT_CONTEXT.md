# Project Context

## 项目信息

- **项目名称**：Petty — 宠物上门喂养服务平台
- **行业**：宠物服务 O2O
- **商业模式**：双边平台（宠物主人 ↔ 喂养师），按单抽佣 + 会员增值
- **目标**：构建一套可交给 AI Agent 持续迭代的宠物上门喂养全流程系统

---

## 用户角色

| 角色 | 说明 |
|------|------|
| 宠物主人（Pet Owner） | 发布服务需求、预约喂养、查看服务过程、评价打赏 |
| 喂养师（Pet Sitter） | 接单服务、上门喂养、拍照打卡、收入结算 |
| 平台运营（Admin） | 审核资质、调度订单、风控预警、数据分析 |

---

## 核心业务流程

```
宠物主人下单 → 智能匹配喂养师 → 喂养师确认接单 → 上门打卡(GPS+照片)
    → 执行服务(喂食/铲屎/陪玩) → 服务报告上传(照片+视频+文字)
    → 主人确认完成 → 评价 → 资金释放(扣佣后打款给喂养师)
```

---

## 核心对象

| 对象 | 说明 |
|------|------|
| Pet | 宠物档案（种类、品种、饮食习惯、健康禁忌、疫苗记录） |
| Owner | 宠物主人（地址、门锁信息、紧急联系人） |
| Sitter | 喂养师（资质认证、服务区域、可用时段、评分） |
| ServiceType | 服务类型（上门喂养、遛狗、陪玩、洗护、寄养） |
| Order | 服务订单（预约时间、服务内容、金额、状态） |
| ServiceLog | 服务记录（打卡照片、GPS轨迹、服务报告） |
| Payment | 支付与结算（托管资金、佣金抽成、提现） |
| Review | 双向评价（主人评喂养师、喂养师评主人） |

---

## 部署目标

- **MVP 阶段**：微信小程序（宠物主人端 + 喂养师端） + 管理后台 Web
- **未来扩展**：iOS/Android 原生 App + 智能设备对接（宠物摄像头、智能门锁）
- **数据存储**：MySQL（核心业务） + Redis（匹配缓存/会话） + OSS（媒体文件）
